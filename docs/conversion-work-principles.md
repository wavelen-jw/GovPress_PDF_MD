# Conversion Work Principles

이 문서는 PDF/HWPX -> Markdown 변환 품질 개선 작업의 공통 원칙을 정리한다.
이후의 분석, 회귀 점검, golden 비교, 규칙 수정은 모두 이 원칙을 따른다.

## 핵심 원칙

1. 문서 전용 예외를 만들지 않는다.
- 특정 파일명
- 특정 문서 제목
- 특정 날짜/기관명
- 특정 샘플에만 맞는 하드코딩

위 방식은 금지한다.

2. 모든 수정은 범용 규칙 추출을 목표로 한다.
- golden과 현재 변환 결과의 차이를 본다.
- 차이를 만든 원인을 구조 패턴으로 설명할 수 있어야 한다.
- 그 패턴이 다른 문서에도 재사용 가능한지 확인한다.

3. golden 비교의 목적은 "정답 복붙"이 아니라 "범용 규칙 발견"이다.
- golden은 문서별 예외를 만들기 위한 자료가 아니다.
- golden은 현재 규칙이 놓치는 구조 패턴을 찾는 데 사용한다.

4. press는 명확한 유형으로 별도 처리한다.
- press 판별 규칙은 유지한다.
- press가 아니면 기본적으로 report 계열로 본다.
- 필요 시 report 내부 하위 패턴을 구분하되, generic으로 회피하지 않는다.

5. 새로운 유형 분리는 구조적으로 반복되는 경우에만 한다.
- 단일 문서만을 위해 새 유형을 만들지 않는다.
- 여러 문서에서 반복되는 구조가 있을 때만 새로운 하위 유형을 검토한다.
- 예: full_report처럼 장문 목차/표지/붙임 구조가 반복될 때

## 우선적으로 추출해야 하는 범용 패턴

1. 비교표 패턴
- 좌측 라벨 열
- 가운데 전환/화살표 열
- 우측 비교 대상 열
- 일부 rowspan/colspan이 있어도 grid 복원이 가능하면 Markdown 표로 우선 시도한다.

2. 참고 블록 패턴
- 참고1, 참고2, 붙임 등 번호형 부속 블록
- 제목 + 하위 소제목 + bullet/표/quote 조합
- heading과 blockquote 구조를 일반화해서 복원한다.

3. 장문 payload 분해 패턴
- 추출 과정에서 여러 구조 요소가 한 줄로 합쳐진 경우
- 예: bullet, quote, 표 머리, 그림 라벨이 한 줄에 결합
- 구조 마커를 기준으로 재분해하는 범용 규칙을 우선 검토한다.

4. 구조용 표 평탄화 패턴
- 제목 박스
- 표지 번호/제목 행
- 연락처/부서/보도시점
- 목차형 표
- 데이터 표가 아닌 구조용 표는 일반 본문 구조로 평탄화한다.

5. 병합셀 완화 패턴
- colspan이 있다고 바로 Tier 3로 보내지 않는다.
- nested table, 도식형 표, 의미 손실이 큰 경우만 Tier 3/4로 본다.
- 부분 병합이지만 grid 복원이 가능하면 Tier 2로 처리한다.

## 작업 절차

1. 새 샘플을 변환한다.
2. golden 또는 PDF/HWPX 결과 차이를 확인한다.
3. 차이를 구조 패턴 단위로 분류한다.
4. 문서명 기반 예외가 아닌 범용 규칙으로 해결 가능한지 검토한다.
5. 규칙을 추가/수정한다.
6. converted 결과를 다시 생성한다.
7. 회귀를 확인한다.

## 금지 사항

1. 파일명 기반 분기
2. 제목 문자열 기반 하드코딩
3. 하나의 샘플만 통과시키는 임시 치환
4. golden 본문을 직접 삽입하는 방식

## 허용되는 예외

아래는 문서 전용 예외가 아니라 구조 패턴이 반복되는 경우에만 허용한다.

1. 특정 목차 구조를 가진 장문 보고서형 문서
2. 반복적으로 등장하는 참고/붙임 블록 구조
3. 반복적으로 등장하는 비교표/전환표/단계표 구조

## 판단 기준

규칙을 추가하기 전에 아래 질문에 "예"라고 답할 수 있어야 한다.

1. 이 규칙은 다른 문서에도 적용될 가능성이 있는가?
2. 이 차이는 추출 문제보다 구조 패턴 문제로 설명 가능한가?
3. 이 규칙을 넣어도 press 경로를 건드리지 않는가?
4. 회귀 검증으로 부작용을 확인할 수 있는가?

## 유지 원칙

이 문서는 이후 변환 품질 개선 작업의 기준 문서다.
새로운 문서가 들어와도 먼저 이 원칙에 따라 분석하고, 필요하면 이 문서를 갱신한다.
