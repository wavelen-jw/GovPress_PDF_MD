# Conversion Rules

이 문서는 현재 프로젝트의 보도자료 PDF → Markdown 변환 규칙만 별도로 정리한 것입니다.

## 파이프라인

1. `opendataloader-pdf`를 `markdown,json` 형식으로 실행합니다.
2. 출력 결과에서 `.json`을 우선 사용합니다.
3. `.json`이 없을 때만 `.md`를 사용합니다.
4. 추출 결과를 보도자료 템플릿 규칙으로 정규화합니다.
5. 최종 Markdown을 편집기와 저장 결과에 사용합니다.

## 문서 구조 인식

- `보도자료`는 문서 라벨로 인식합니다.
- `보도시점`은 메타데이터로 분리합니다.
- 여러 줄 제목은 하나의 `#` 제목으로 병합합니다.
- 제목 아래 `-` 요약문은 bullet로 유지합니다.
- 본문과 담당부서를 분리합니다.
- `□`가 없는 일반형 보도자료는 제목/부제 뒤 첫 일반 서술문부터 본문으로 인식합니다.

## 본문 규칙

- `□`는 항상 일반 문단으로 처리합니다.
- `○`, `ㅇ`는 기본 bullet로 처리합니다.
- `-`, `*`는 하위 bullet 또는 설명 주석으로 처리합니다.
- `<...>` 형식 라벨은 `#### <...>` 소제목으로 처리합니다.
- 문장 뒤에 붙은 `<...>` 라벨은 문장과 분리합니다.
- 설명형 문장은 heading으로 승격하지 않습니다.
- `△` 하위 항목은 앞 bullet의 하위 목록으로 유지합니다.

## 참고와 주석

- `<참고 : ...>`는 blockquote로 유지합니다.
- 참고/연혁 블록은 blockquote로 유지합니다.
- bullet 바로 아래에 붙는 설명성 `*` 문구는 들여쓴 blockquote로 처리할 수 있습니다.
- 필요한 경우 각주성 설명은 `  > ...` 형태로 정리합니다.
- bullet 바로 아래에 나오는 상자형 안내 블록은 들여쓴 quote 블록으로 유지할 수 있습니다.

## 표와 상자 규칙

- 연락처 표는 별도 연락처 처리 규칙을 우선 적용합니다.
- 비연락처 표 중 `1행 1열` 표는 글상자형 안내 박스로 보고 quote 블록으로 렌더링합니다.
- 단순 표는 Markdown table로 변환합니다.
- 병합 셀(`rowspan`, `colspan`)이 있는 복잡한 표는 HTML table로 fallback 합니다.
- 표 셀 안의 `§`, `•`, `‧` 연속 항목은 `<br>`로 분리합니다.
- 표 셀 안의 `§`는 출력 시 `•`로 치환합니다.

## 담당부서 규칙

- 연락처 표는 부서, 책임자, 담당자를 연결해 복원합니다.
- `rowspan`이 있는 표도 부서 정보가 다음 행에 이어지도록 처리합니다.
- 최종 출력은 `## 담당부서` 아래 bullet 목록입니다.
- 연락처 뒤에 붙는 `참 고`, `붙임`, `붙 임`, `별첨` 등 후행 노이즈는 제거합니다.

## 제거 규칙

- 페이지 번호 같은 노이즈는 제거합니다.
- 이미지 노이즈와 목차 잔재는 제거합니다.
- `붙임`, `붙 임`, `별첨`은 최종 Markdown에 포함하지 않습니다.
- `:Zone.Identifier` 같은 Windows 메타데이터 파일은 변환 대상에서 제외합니다.

## 적용 범위

- 현재 규칙은 행정기관 보도자료 템플릿을 기준으로 최적화되어 있습니다.
- 동일 문서군 안에서도 형식 차이가 큰 보도자료는 별도 템플릿으로 분리하는 것이 더 안전할 수 있습니다.
- 일반 문서나 스캔 PDF는 동일한 품질을 보장하지 않습니다.

## 배포 메모

- 비개발자용 Windows 배포는 번들된 Python 런타임과 Java 런타임을 포함하는 방식을 전제로 합니다.
- Java 런타임은 별도 설치를 요구하지 않도록 `runtime/java` 아래에 포함할 수 있습니다.
- 배포용 Java는 번들 가능한 OpenJDK 계열을 사용해야 하며, 현재 권장 기준은 Temurin 17입니다.
